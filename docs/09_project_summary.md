# 09. Project Summary

## 1. Key Findings

A casual / fishing-style game monetization system is built around the balance between:

- Short-session gameplay
- Continuous resource consumption
- Variable rewards
- Resource shortage
- Optional recovery through ads, bundles, or recharge
- Event-driven motivation
- User segment-specific goals

The core challenge is not simply creating payment pressure. The system must create different levels of achievable goals for different user segments.

## 2. Economy Insight

Coins and ammo are the main resource sink. If consumption is too low, payment motivation weakens. If consumption is too high, F2P and low-spending users may churn.

A healthier structure provides:

- Basic continuation for F2P
- Efficient recovery for ad users
- Clear value for low spenders
- Achievable event goals for mid spenders
- Competition and status for heavy spenders

## 3. Monetization Insight

Paid bundles work best when they match real player states:

| Player State | Suitable Monetization |
|---|---|
| New user | Starter pack, first purchase |
| Daily active user | Daily pack, pass EXP |
| Resource shortage | Bankruptcy pack, rewarded ads |
| Event participant | Event pack, token supplement |
| Competitive payer | Ranking pack, cumulative recharge |
| Returning user | Return pack, catch-up reward |

The same bundle shown to all users is less effective than a bundle triggered by context.

## 4. Live-Ops Insight

A 30-day cycle should balance:

- Onboarding
- Daily routine
- Weekly activity
- Festival events
- Leaderboards
- Recharge events
- Returning user recovery
- Pass completion

Revenue peaks should be evaluated together with retention and resource health.

## 5. Metrics Insight

Commercialization analysis should connect:

- DAU and retention
- Session and resource consumption
- Resource gap and monetization triggers
- Ad revenue and IAP revenue
- Payer rate and ARPPU
- LTV and ROAS
- Event participation and post-event retention

A short-term payer rate increase is not always positive if it is accompanied by lower retention, higher bankruptcy rate, or reduced mid-spender satisfaction.

## 6. Final Conclusion

This project demonstrates a structured analysis framework for casual / fishing-style monetization systems.

The main analytical value lies in connecting:

- Player motivation
- System purpose
- Resource economy
- Paid triggers
- User segmentation
- Live-ops rhythm
- KPI diagnosis
- Tuning recommendations

The project uses simulated data honestly and explicitly, focusing on analytical reasoning rather than claiming access to real product data.
