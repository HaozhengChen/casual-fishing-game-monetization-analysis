# Limited-time Event Analysis: Deep Sea Treasure Boss Challenge

## 1. Event Design

The simulated event is a 5-day Boss challenge designed for a casual / fishing-style game.

The event adds:

- Event tickets
- Boss Room entries
- Event tokens
- Event shop
- Leaderboard
- Event packs
- Cumulative recharge rewards

The core design intention is to increase DAU, deepen activity participation, create resource consumption, and improve payer conversion.

## 3. Diagnosis

The event successfully increased activity and payer conversion, but it also introduced retention risk.

The most likely issue is that the event resource pressure was too aggressive for F2P and mid-spender segments.

### 3.1 DAU Up but D1 Retention Down

This suggests that the event attracted users but reduced next-day motivation for part of the player base.

Possible causes:

- High event cost
- Fast coin consumption
- Weak participation reward
- Overly top-heavy leaderboard
- Lack of achievable mid-tier goals

### 3.2 Payer Rate Up but ARPPU Flat

This suggests that more users made low-value purchases, but high-value spending did not increase proportionally.

Possible causes:

- Bankruptcy packs converted low spenders
- Event packs were not strong enough for heavy spenders
- Mid spenders lacked confidence in event completion

### 3.3 Ad Watch Rate Up but Bankruptcy Rate Up

This suggests that users wanted to continue but free recovery was insufficient.

Possible causes:

- Ad reward too low relative to Boss Room cost
- Ticket supply too limited
- No sufficient coin return floor

## 4. Tuning Plan

### 4.1 Lower Early Event Pressure

- Daily first 3 Boss Room entries use lower consumption.
- Add daily free tickets through tasks.
- Add participation floor reward.
- Keep high-cost challenge as optional advanced mode.

### 4.2 Add Milestone Rewards

- Personal token milestones
- Daily participation milestones
- Mid-tier completion path
- Cosmetic rewards for higher investment

### 4.3 Redesign Leaderboard

- Add grouped leaderboards
- Add participation and milestone rewards
- Keep top rewards more cosmetic
- Reduce direct power gap from ranking rewards

### 4.4 Improve Ad Supply

- First few daily ad rewards provide meaningful recovery.
- Later ad rewards use diminishing returns.
- Ads can provide tickets but not replace premium event packs.
- Boss failure can trigger one ad-based retry.

## 5. Final Assessment

The event is directionally effective but economically too aggressive.

The main adjustment should not remove monetization pressure entirely. Instead, it should segment the pressure:

- F2P: basic participation
- Ad users: limited continuation
- Low spenders: clear value recovery
- Mid spenders: achievable main reward
- Heavy spenders: optional competition and status

A healthier event should increase revenue without causing large-scale resource exhaustion or lower next-day retention.
